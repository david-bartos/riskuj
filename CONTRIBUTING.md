# Contributing

Thanks for helping improve Riskuj.

## Opening issues

Use issues for bug reports, feature requests, and collaboration proposals.
Security vulnerabilities should be reported privately as described in
`SECURITY.md`.

## Pull requests

Pull requests from forks are welcome. The `main` branch is protected, so every
change must go through a pull request and receive maintainer review before it
can be merged.

Before opening a pull request:

1. Run `npm run build`.
2. Run `npm test` when changing game logic, imports, storage, API behavior, or
   UI flows with tests.
3. Keep generated/local files out of the commit, especially `node_modules/`,
   `dist/`, `.env`, and runtime files in `data/`.
